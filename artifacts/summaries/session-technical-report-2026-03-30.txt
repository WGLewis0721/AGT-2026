PROMPT 02 — PRICING LAMBDA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OBJECTIVE
Create backend/lambda/pricing_lambda.py — a standalone TRA3 microservice that
validates order selections, recalculates price server-side (browser amounts
intentionally ignored), and creates a Stripe Checkout Session for the deposit.

FILE: backend-integration/lambda/pricing_lambda.py  (NEW — 215 lines)

  Constants (server-side source of truth):
  · PACKAGES — 3 entries, same prices as frontend
  · ADDONS   — 10 entries, same prices as frontend
  · DEPOSIT_RATE = 0.20

  _response(status, body, origin)
  · Builds API Gateway response envelope
  · CORS headers: Access-Control-Allow-Origin, Allow-Headers, Allow-Methods
  · Dev/localhost origin bypass for local testing

  _calculate_price(package_key, addon_keys)
  · Returns { total, deposit, balance, deposit_cents }
  · deposit_cents = round(total * DEPOSIT_RATE * 100) — used for Stripe unit_amount
  · Browser-submitted amounts are never trusted

  _create_checkout_session(package_key, addon_keys, origin)
  · Calls stripe.checkout.Session.create()
  · mode: "payment", unit_amount: deposit_cents, currency: "usd"
  · success_url: {origin}/success.html?session_id={CHECKOUT_SESSION_ID}
  · cancel_url:  {origin}/#booking
  · metadata: package, addons (comma-sep), total, deposit, balance, cal_url,
              client, environment

  lambda_handler(event, context)
  · OPTIONS → 200 CORS preflight
  · POST    → parse body, validate package_key, call _calculate_price +
               _create_checkout_session, return { url } to frontend
  · Structured JSON logging to CloudWatch for every request

  Validation: python -m py_compile → OK
