================================================================================
SECTION B — TOP 3 RISKS (Plain English)
================================================================================

1. **State Lock Missing (HIGH)**: The Terraform S3 backend has no DynamoDB lock
   table configured, which means if two engineers deploy simultaneously, the
   second deployment can corrupt the state file, leading to orphaned AWS resources,
   inconsistent infrastructure state, or complete inability to manage resources
   via Terraform until state is manually recovered.

2. **API Gateway Not Throttled (MEDIUM)**: The POST /webhook endpoint has no
   rate limiting configured, which means a malicious actor who discovers the
   webhook URL could spam it with requests to rack up Lambda invocation costs
   and trigger excessive SMS sends to the detailer's phone, potentially costing
   hundreds of dollars and causing service disruption.

3. **Service Price Matching Logic Flaw (MEDIUM)**: The SERVICE_PRICES matching
   logic uses substring containment (`if key in service_lower`) which processes
   keys in dictionary iteration order - if a customer somehow submits a service
   name containing multiple size keywords, it would match the first occurrence
   rather than the most specific match, potentially causing incorrect pricing
   calculations and revenue loss.
