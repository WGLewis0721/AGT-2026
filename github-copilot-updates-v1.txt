PROMPT 02 — LAMBDA FUNCTION
Path: backend-integration/lambda/ and backend-integration/layer/

    stripe==7.10.0
    requests==2.31.0

  layer/requirements.txt
    stripe==7.10.0
    requests==2.31.0

  lambda/lambda_function.py
    Full Stripe webhook handler — 11 steps:

    Step 1  Extract body and stripe-signature header; set stripe.api_key
    Step 2  Verify Stripe signature via stripe.Webhook.construct_event()
            · SignatureVerificationError → log + return 400 "Invalid signature"
            · Any other exception        → log + return 400 "Webhook error"
    Step 3  Log stripe_webhook_received (structured JSON to CloudWatch)
    Step 4  Filter: ignore all event types except checkout.session.completed
    Step 5  Extract session and customer details (name, email, phone, deposit)
    Step 6  Extract custom fields: service, date, location
    Step 7  Calculate balance due using SERVICE_PRICES substring map:
              "sm detail" → $100, "md detail" → $150, "lg detail" → $200
              "small" → $100, "medium" → $150, "large" → $200
              "full detail" → $150
            Log balance_calculated event
    Step 8  Send detailer SMS (CRITICAL — returns 500 if delivery fails)
            · Includes: Name, Phone, Email, Service, Date, Location,
              Deposit, Balance (omitted if service not in price map)
            · No URLs in message body
    Step 9  Send customer SMS (NON-CRITICAL — 500 never returned on failure)
            · Skipped gracefully if no phone on file
            · Includes: confirmation, service, date, location, deposit,
              balance (or "Balance collected after service."), business phone
            · Failure logged but booking is still considered confirmed
    Step 10 Log booking_processed summary (environment, customer_name,
            service, deposit_paid, balance_due, detailer_sms, customer_sms)
    Step 11 Return 200 "Booking processed"

    Helpers:
      _response(status_code, message) — standard API GW response dict
      _send_sms(phone, message, recipient_label) — Textbelt POST, never raises

    Module-level constants:
      STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, TEXTBELT_API_KEY,
      DETAILER_PHONE, ENVIRONMENT — all from os.environ
      SERVICE_PRICES dict, BUSINESS_PHONE = "(334) 294-8228"

    Security rules honoured:
      · Sensitive env vars never logged
      · Full phone numbers never logged
      · All print() calls use json.dumps structured logs
      · Bracket access used on stripe_event and stripe_event["data"]["object"]
      · .get() with fallbacks used on session fields (nullable in test events)

PROMPT 03 — DEPLOY SCRIPTS
Path: backend-integration/scripts/
────────────────────────────────────────────────────────────────────────────────
Files created (2):

  bootstrap-layer.ps1
    Run once per AWS account; re-run when layer deps change.
    $ErrorActionPreference = "Stop" set at top.
    5-step banner output with timing:

      [STEP 1/5] Get AWS account ID (aws sts get-caller-identity)
                 Derive S3 bucket name: tra3-{account_id}-deployments
      [STEP 2/5] Create build directory (backend-integration\layer-build\python\)
                 Cleans previous build if present
      [STEP 3/5] pip install layer/requirements.txt → build dir
                 Full output visible (no --quiet)
      [STEP 4/5] Compress-Archive build dir → backend-integration\layer.zip
                 Prints zip size in KB
      [STEP 5/5] aws s3 cp → s3://{bucket}/layers/dependencies/layer.zip
                 Cleans up build dir and zip after upload

    Every command printed as DarkGray before execution.

  deploy.ps1
    Full infrastructure deploy. Run from repo root.
    Parameters:
      -Client       [Mandatory]  client slug, e.g. gentlemens-touch
      -Environment  [Optional]   "dev" or "prod" (default: "prod")
      -Force        [Switch]     forces terraform reinit

    $ErrorActionPreference = "Stop" set at top.
    5-step banner output:

      [STEP 0/5] Preflight checks
                 · terraform --version
                 · aws --version
                 · aws sts get-caller-identity (derive account ID + S3 bucket)
                 · Verify tfvars file exists (exits 1 with helpful message if not)
                 · Verify layer.zip present in S3 (exits 1 if not, points to
                   bootstrap-layer.ps1)
      [STEP 1/5] Package Lambda
                 · Compress lambda_function.py → lambda_function.zip
                 · aws s3 cp → functions/{client}/{env}/lambda_function.zip
      [STEP 2/5] Terraform Init
                 · Workspace: dev → named "dev" workspace; prod → "default"
                 · terraform init -backend-config (bucket, key, region)
                   skipped if .terraform/ exists and -Force not set
      [STEP 3/5] Terraform Apply
                 · terraform apply -auto-approve with client_name, environment,
                   and var-file pointing to clients/{client}/{env}.tfvars
      [STEP 4/5] Outputs
                 · terraform output -json | ConvertFrom-Json
                 · Prints webhook URL prominently
      [STEP 5/5] Next Steps
                 · Instructions for Stripe dashboard webhook setup
                 · dev: toggle to TEST mode; prod: toggle to LIVE mode
                 · Reminds to re-run script after adding webhook signing secret
