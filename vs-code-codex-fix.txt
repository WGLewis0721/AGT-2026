Problems solved

1. Cal.com bookings were not triggering the Lambda correctly
- Root cause: Cal.com uses its own Stripe integration and sends Payment Intent based booking webhooks, not Stripe checkout.session.completed events.
- Fix: Added direct Cal.com webhook handling for BOOKING_PAYMENT_INITIATED.

2. Stripe and Cal.com needed to share one endpoint
- Root cause: both services hit the same API Gateway/Lambda URL.
- Fix: Added routing logic so Stripe requests still use Stripe signature verification and Cal.com requests use Cal.com signature verification.

3. Cal.com webhook signature validation was missing
- Root cause: no support existed for x-cal-signature-256 verification.
- Fix: Added HMAC verification using CALCOM_WEBHOOK_SECRET.

4. Booking details from Cal.com needed to be normalized for SMS
- Root cause: Cal.com payload shape is different from Stripe checkout session payloads.
- Fix: Added parser logic to extract service, add-ons, phone, time, and payment amount into the same kind of booking summary used by SMS logic.

5. Terraform did not know about the Cal.com secret
- Root cause: CALCOM_WEBHOOK_SECRET was not defined as a variable or Lambda environment value.
- Fix: Added Terraform variable and environment wiring, then redeployed.

6. Packaging instructions did not match the repo's actual deployment path
- Root cause: the request referenced booking-lambda.zip, but Terraform and the deployment flow actually use lambda_function.zip stored in S3.
- Fix: rebuilt both zip files and deployed using the existing S3-backed lambda_function.zip path so AWS picked up the new code.

7. Needed proof that Stripe routing was not broken
- Fix: sent a signed test Stripe-style event to the dev webhook and confirmed stripe_webhook_received in CloudWatch.

8. Needed proof that Cal.com routing and signature verification worked in prod
- Fix: sent a signed safe Cal.com probe and confirmed:
  - calcom_webhook_received
  - calcom_ignored

9. Needed proof that end-to-end Cal.com booking SMS flow worked
- Fix: sent a signed BOOKING_PAYMENT_INITIATED test event to prod and confirmed in CloudWatch:
  - calcom_webhook_received
  - calcom_booking_parsed
  - balance_calculated
  - detailer_sms_sent
  - customer_sms_sent
  - calcom_booking_processed
