Click .btn-package or
  .btn-gold in booking funnel → openBookingModal(calUrl, stripeUrl)
                                  pendingCalUrl    = calUrl
                                  pendingStripeUrl = stripeUrl || ''
                                  #waiver-modal.classList.add('open')
                                  document.body.style.overflow = 'hidden'

Click .addon-card            → toggleAddon(el)
                                  el.classList.toggle('selected')
                                  (visual only; not persisted anywhere)

Click #waiver-agree-btn      → agreeAndBook()
                                  window.open(pendingCalUrl, '_blank')  // Cal.com
                                  if (pendingStripeUrl)
                                    window.location.href = stripeUrl    // Stripe
                                  close modal + clear state

Click .waiver-close or
  modal backdrop             → closeWaiverModal()
                                  modal.classList.remove('open')
                                  pendingCalUrl = pendingStripeUrl = ''

Footer "Waiver & Terms" link → openWaiverOnly()
                                  pendingCalUrl = '' (agree btn hidden)
                                  modal opens in read-only mode

CRITICAL GAP: Selected addons (.addon-card.selected) are NEVER read or
forwarded to Cal.com / Stripe. There is no booking object anywhere in the
current JS — only two module-level strings (pendingCalUrl, pendingStripeUrl).

3. #waiver-agree-btn calls agreeAndBook() which does:
     window.open(pendingCalUrl, '_blank')
     if (pendingStripeUrl) window.location.href = pendingStripeUrl
4. Addon cards (.addon-card) in section#addons toggle class 'selected' via
   toggleAddon(el). Selected addons are NEVER read or forwarded. This is a gap.
5. No structured booking object exists. State = two module-level strings.

     data-stripe-url="<existing stripeUrl or empty>"
